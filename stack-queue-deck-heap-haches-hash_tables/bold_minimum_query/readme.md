## Дерзкий запрос минимума
ограничение по времени на тест: 5 секунд  
ограничение по памяти на тест: 512 мегабайт  

Преподаватель группы B1 Рузаль предложил использовать для олимпиады следующую задачу:

Вам дан массив из n целых чисел, где n может быть до $10^7$, научитесь выполнять запросы «присвоить i-му элементу значение x» и сообщать значение глобального минимума в массиве.

Рузаль очень опытный разработчик задач. Он знает, что случайные тесты прекрасно проверят любое решение и нет никакой необходимости добавлять какие-либо ещё тесты. Однако, олимпиада делается в последнюю ночь, и Рузаль слишком занят просмотром видео на YouTube, поэтому он придумал гениальное решение: вы будете генерировать себе тесты самостоятельно.

Вам дан линейный конгруэнтный генератор псевдослучайных чисел, определяемый целыми числами $x_0, x_1$, a, b и c. Ниже приведён код процедуры генерации чисел на языке C++:


uint32_t next() {  
 uint32_t t = x0 * a + x1 * b + c;  
 x0 = x1;  
 x1 = t;  
 return x0 » 2;  
}

Здесь означает uint32_t беззнаковый 32-битный целочисленный тип данных. Переполнение обрабатывается взятием по модулю $2^{32}$ (после выполнения каждой операции от числа остаются младшие 32 бита).

Аналогичная процедура для языка Java (обратите внимание, хотя int в языке Java является знаковым типом, переполнение правильно обрабатывается как взятие по модулю $2^{32}$):


int next() {  
 int t = x0 * a + x1 * b + c; 
 x0 = x1;  
 x1 = t;  
 return x0 »> 2;  
}

Изначально все элементы массива содержат значение $2^{31}-1$. Для каждого запроса двумя последовательными вызовами функции next() генерируются числа i и x, то есть за вызовом "i = next()" следует вызов "x = next()". Такая пара определяет запрос "a[i % n] = x".

Обозначим минимум после выполнения i-го запроса как $s_i$. Вам требуется найти значение $\sum\limits_{i=1}^q s_i * 10099^i \mod 2^{32}$

Обратите внимание, единственной целью указанного выше генератора является приближение случайного генератора. Для решения задачи от вас не требуется изучать и использовать какие-либо его свойства. Параметры генератора будут выбираться чтобы сделать тесты случайными насколько это возможно.

**Входные данные**  
В единственной строке входных данных записаны целые числа n, q, $x_0, x_1$, a, b, c ($1\leq n\leq10^7$, $1\leq q\leq 5·10^7$, $0\leq$$x_0,x_1,c<2^31,2\leq a,b<2^31$, a и b являются нечётными). Здесь n означает размер массива, q — количество запросов, остальные числа являются параметрами псевдослучайного генератора.

**Выходные данные**  
Выведите одно целое число: значение $\sum\limits_{i=1}^q s_i * 10099^i \mod 2^{32}$.
